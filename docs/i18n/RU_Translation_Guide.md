## Правила перевода (i18n Quality Rules v1)

### Основные требования:

1. **Второе лицо (ты)**
   - Используй "ты" (неформальное обращение)
   - Пример: "Ты склонен обрабатывать эмоции тихо..."

2. **Вероятностный язык**
   - Избегай категоричности (always, never, must, should)
   - Используй: может, склонен, часто, обычно
   - Пример: "Ты можешь заметить..." вместо "Ты должен заметить..."

3. **Спокойный, заземленный тон**
   - Избегай драматизации
   - Избегай психологизирующих формулировок
   - Фокус на наблюдении, а не интерпретации

4. **Длина текста**
   - Рекомендуемая: 180-320 символов
   - Допустимо: ±20% (144-384 символов)

5. **Запрещенные слова**
   - must, should, have to, you need to → избегать
   - always, never → избегать
   - disorder, trauma, pathology → избегать
   - destiny, fate, karma, meant to → избегать

## Примеры правильных переводов

### Observation (OBS)
```
EN: "You tend to process emotions quietly, often noticing them more clearly once you have a bit of space."
RU: "Ты склонен обрабатывать эмоции тихо, часто замечая их яснее, когда появляется немного пространства."
```

### Interpretation (INT)
```
EN: "Feelings often stack quietly before you name them."
RU: "Чувства часто накапливаются тихо, прежде чем ты их назовёшь."
```

### Context (CON)
```
EN: "You may notice that signal your tempo so others know the pause is intentional."
RU: "Ты можешь заметить, что сигнализируешь свой темп, чтобы другие знали, что пауза намеренна."
```

## Процесс заполнения

### Вариант 1: Использование Translation API

1. **DeepL API** (рекомендуется для качества)
   ```python
   # Пример интеграции с DeepL
   import deepl
   translator = deepl.Translator(api_key)
   ru_text = translator.translate_text(en_text, target_lang="RU")
   ```

2. **OpenAI API**
   ```python
   # Пример с GPT-4 для контекстуального перевода
   response = openai.ChatCompletion.create(
       model="gpt-4",
       messages=[{
           "role": "system",
           "content": "Ты переводчик, следуешь правилам i18n quality rules v1..."
       }, {
           "role": "user",
           "content": en_text
       }]
   )
   ```

### Вариант 2: Ручной перевод

1. Открой `CONTENT/i18n/en.json` и `CONTENT/i18n/ru.json`
2. Для каждого ключа:
   - Прочитай EN текст
   - Переведи следуя правилам выше
   - Проверь длину (180-320 символов)
   - Проверь отсутствие запрещенных слов
   - Запиши в ru.json

### Вариант 3: Пакетная обработка

Используй скрипт `scripts/generate_ru_with_llm.py` как основу для интеграции с translation API.

## Проверка качества

После заполнения переводов запусти валидацию:

```bash
python3 scripts/validate_i18n.py
```

Это проверит:
- Структурную валидность (все ключи присутствуют)
- Стиль (отсутствие запрещенных слов)
- Длину текстов
- Соответствие правилам
