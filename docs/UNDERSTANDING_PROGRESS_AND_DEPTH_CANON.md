## Главный принцип

> **Прогресс TodayFlow определяется не количеством открытых функций, а ростом качества понимания пользователя.**

Из этого следуют все решения ниже: нет пустых экранов; нет искусственно обрезанных блоков; отсутствующие данные всегда с объяснением ценности; подписка увеличивает **глубину анализа**, а не прячет «основную часть продукта»; Profile (и Personal Model) богатеет по мере подтверждённых данных и взаимодействия.

Companion ([PR4](./PR4_PROFILE_CANON.md) / [EXPLAIN_MEANING](./EXPLAIN_MEANING_NOT_MECHANISM.md)):

> Интерфейс показывает **не максимум данных**, а **максимум понимания**.

## 1. Никогда не «пусто»

Запрещено состояние продукта:

> Нет данных.

Обязательное состояние:

> Уже определено **вот это**. Для следующего уровня понимания нужны ещё **вот эти** данные — и вот **зачем**.

Примеры (Profile / natal):

| Есть | Уже можно | Без чего | Откроется |
|------|-----------|----------|-----------|
| Дата рождения | нумерология · положения планет · базовые архетипы · значимая часть портрета | — | — |
| + время рождения | — | ASC · дома · более точные интерпретации личности · точнее персональные рекомендации | асцендент, дома, глубина портрета |
| + место / точность координат | точнее дома/ASC при известном времени | — | точность расчёта |

Не ошибка. Не placeholder. Не «весь профиль недоступен».  
Честное объяснение ценности следующего шага.

Паспорт блока ([PRODUCT_TRUTH_FIRST](./PRODUCT_TRUTH_FIRST.md)): состояние `partial` + **одно** конкретное действие + **зачем**.

## 2. Отсутствующее поле отвечает на три вопроса

Любой missing-data блок и CTA обязаны дать:

1. **Что отсутствует** (о данных человека, не о «системе»).  
2. **На что влияет / что откроется.**  
3. **Как исправить** — CTA с **ценностью действия**, не с фактом пустоты.

Канон голоса: [TODAYFLOW_VOICE_CANON.md](./content/TODAYFLOW_VOICE_CANON.md) §0.05–§0.06.

| ❌ | ✅ |
|----|----|
| Добавить время рождения. | Добавьте время рождения, чтобы открыть Асцендент и дома. |
| Нам не хватает наблюдений. / Недостаточно данных. | Повторяющиеся жизненные закономерности проявляются со временем. → Перейти к Today |
| Профиль готов / формируется. | (не говорить) — показать доступное о человеке + следующий смысл |

Exemplars:

> Время рождения не указано.  
> Асцендент и дома зависят от времени рождения.  
> Добавьте время рождения, чтобы увидеть более точную структуру натальной карты.

> Место рождения не указано.  
> Место рождения влияет на положение домов в натальной карте.  
> Добавьте место рождения, чтобы сделать расчёт точнее.

> Повторяющиеся жизненные закономерности проявляются со временем.  
> Продолжайте отмечать своё состояние, чтобы увидеть, какие решения, эмоции и события действительно повторяются именно в вашей жизни.  
> → Перейти к Today

Нет «просто заполните поле» без ценности. Нет отчёта о том, что «системе не хватает».

## 3. Отсутствие данных влияет только там, где влияет

| Нет | Недоступно (честно omit) | Всё ещё доступно |
|-----|--------------------------|------------------|
| Дата рождения | натал · нумерология · персональная совместимость по карте | дневник · цели · check-in · общий Today без персонального натала |
| Время рождения | ASC · дома · производные, требующие горизонта | планеты · нумерология · многие архетипы · значительная часть Profile |
| Место / координаты | точность домов/ASC | всё, что не зависит от геолокации |
| Подписка (после trial) | **глубина** (почему / как проявляется / как меняется / longitudinal evidence) | **наличие** Profile и базовый ответ «кто ты» |

Запрет: «весь раздел закрыт», если закрыт только один расчётный слой.

---

## 4. Подписка меняет глубину, не наличие

Подписка **не** прячет Profile и **не** режет экраны на «обрезанные блоки».

| | Без подписки (и в trial) | С подпиской |
|--|-------------------------|-------------|
| Profile отвечает | **Кто ты?** | **Кто ты?** + **почему именно так?** · **как проявляется?** · **как меняется?** · **что подтверждает за месяцы?** |
| Today | Целостный день | + более глубокая персонализация и память |
| Модель | Базовый Snapshot | Живая модель: гипотезы подтверждаются / опровергаются evidence |

Запрет UX:

- серые вырезанные карточки «только в Premium»;
- половина экрана с замками без целостного опыта;
- ощущение, что продукт спрятали, а не что понимание углубилось.

Допустим: честный переход «углубить разбор» с ясной ценностью следующего уровня (как missing-field CTA).

---

## 5. Trial — полноценный опыт

Trial действует **до** устойчивого ежедневного возврата и накопления подтверждённых наблюдений (сигнал: человек заходит регулярно, портрет и день уже полезны).

Trial **обязан** ощущаться как:

> Целостный опыт уже есть. С каждым закрытым днём портрет становится точнее.

Trial **запрещено** ощущаться как:

> Всё закрыто. Заплати, чтобы увидеть продукт.

Пока trial активен — глубина как у полноценного доступа в рамках известных данных (partial по фактам рождения всё равно честный).

---

## 6. Главная ценность подписки

Человек платит **не** за дополнительные вкладки.

Он платит за то, что со временем:

- накапливается **память** о повторяющихся выборах;
- становятся видны **закономерности**;
- гипотезы о личности **подтверждаются или опровергаются**;
- портрет и день становятся **точнее**.

Ценность растёт, потому что Profile / Personal Model становится **живой моделью личности** на накопленных данных — не потому что открылась новая страница.

Связь с [PROFILE_CONTENT_CANON_V1.md](./PROFILE_CONTENT_CANON_V1.md) Reference Rate / Longitudinal Validation: подписка монетизирует рост качества понимания, не feature-count.

## 7. Content plan — глубина по доступу (черновик канона)

Не финальный прайс. **Продуктовый каркас** для контента Profile и соседних проекций.

### A. Зарегистрированный · trial / free · известные факты

| Слой | Показывать |
|------|------------|
| Identity | archetype, sun/moon (если есть), life path, element — labels из расчёта |
| Interpretation | core summary, strengths, decision style, stable helps — по `source_depth` |
| Evidence | honesty: на чём держится портрет; что уточнит |
| Deep Sources | якоря + «что откроет время/место»; полная карта — progressive, без фейка ASC |
| Longitudinal | лёгкий тизер «с закрытыми днями закономерности становятся яснее» — без выдуманной истории |

### B. Подписка · накопленные данные

| Слой | Дополнительно |
|------|----------------|
| Interpretation | how it shows up · patterns · spheres depth |
| Evidence | confirmed themes · 30d/months shift · hypothesis → confidence |
| Deep Sources | акценты домов/аспектов, **реально** вошедшие в портрет |
| Living model | slow evolution (отдельный срез после Source consumption) |

Модули (Today, Compat, Tarot) читают тот же Snapshot; подписка расширяет **глубину проекции**, не invents второй личности.

## 8. Антипаттерны

```text
❌ пустой Profile / «нет данных» / «недостаточно данных»
❌ «мы знаем / система видит / ИИ считает / нам не хватает»
❌ «профиль готов / сформирован / анализ продолжается»
❌ весь раздел закрыт из‑за одного missing field
❌ «укажите время» без ценности («чтобы открыть ASC и дома»)
❌ серые обрезанные блоки Premium
❌ trial = всё за замком
❌ подписка = +N экранов без роста понимания
❌ показывать ASC/дома при time_unknown
```

```text
✅ partial: что отсутствует · что откроется · CTA с ценностью
✅ субъект = человек / смысл / следующий шаг ([Voice Canon §0–§0.06](./content/TODAYFLOW_VOICE_CANON.md))
✅ подписка → глубина «почему / как / что подтверждает»
✅ trial → целостный опыт, затем рост точности портрета
✅ UI = максимум понимания, кухня = полный расчёт
```
