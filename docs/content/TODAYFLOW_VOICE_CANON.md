## 0. TodayFlow не говорит о себе (жёсткий запрет)

> **TodayFlow не рассказывает про себя.**  
> Пользователь не читает про систему, модель, ИИ, алгоритмы или внутренние процессы.  
> Интерфейс объясняет **человека**, **смысл данных** и **ценность следующего шага** — не состояние продукта.

Связь с [EXPLAIN_MEANING_NOT_MECHANISM.md](../explainability/EXPLAIN_MEANING_NOT_MECHANISM.md): смысл и честные границы — да; кухня и «кто посчитал» — нет.

### Запрещённые паттерны (весь продукт · UI · LLM · email · push)

| ❌ Запрещено | Почему |
|-------------|--------|
| «Мы знаем…» / «Мы уже видим…» | продукт как субъект |
| «Мы определили…» / «Мы выяснили…» / «Мы рассчитали…» / «Мы нашли…» | CRM / движок на сцене |
| «Система видит / знает / считает / не может…» | mechanism |
| «ИИ считает…» / «модель считает…» | ломает режим self-discovery |
| «Алгоритм…» / «На основе анализа мы…» | кухня вместо смысла |
| «Нам не хватает…» / «Недостаточно данных…» | взгляд системы, не человека |
| «Профиль готов / сформирован / базовый уровень» | статус пайплайна |
| «Анализ завершён / продолжается» · «Расчёты выполнены» | отчёт о работе продукта |
| «Поможем тебе узнать…» от лица платформы как героя | продукт хвалит себя |

### Замены (человек / смысл / ценность как субъект)

| ❌ | ✅ |
|----|----|
| Мы уже знаем ваши сильные стороны. | Основные сильные стороны уже читаются в портрете. |
| Мы видим повторяющийся паттерн. | Некоторые решения повторяются достаточно часто, чтобы показать устойчивую закономерность. |
| Система не может определить асцендент. | Асцендент и дома зависят от времени рождения. |
| Нам не хватает наблюдений. / Недостаточно данных. | Повторяющиеся жизненные закономерности проявляются со временем. |
| Профиль ещё формируется. / живые тексты / после генерации | Первые контуры характера уже читаются. Повторяющиеся опоры проясняются через отмеченные дни. |
| Мы сможем построить более точный профиль. | Время рождения открывает Асцендент, дома и более точную структуру натальной карты. |

**Правило проверки:** если убрать бренд — текст всё ещё о **человеке и том, что откроется ему**, а не о том, что «система знает / не знает / считает».

## 0.05 Человек, не система (продуктовый голос · весь TodayFlow)

> **Ни один пользовательский текст не описывает состояние системы.**  
> **Любой текст в интерфейсе описывает только пользователя, смысл доступной информации или ценность следующего действия.**

Это правило для **всего** TodayFlow (Profile · Today · Compatibility · Tarot · Goals · Practices · email · push), не только Profile.

Пользователю **не должно быть видно**:

- что знает система;
- чего система не знает;
- чего системе не хватает;
- что система рассчитала / ещё анализирует.

Это внутреннее состояние продукта.

Экран отвечает только на вопросы человека:

1. Что это говорит **обо мне**?  
2. Что ещё может **стать доступно**?  
3. Почему это **важно**?  
4. Что изменится после **следующего шага**?

### Profile (частное следствие)

> Profile никогда не описывает состояние системы.  
> Profile всегда описывает человека, смысл данных и ценность следующего шага.

## 0.06 Missing data и CTA — три обязательных смысла

Сообщаем не «статус заполненности», а:

1. **Что отсутствует** (факт о человеке / его данных).  
2. **На что это влияет** / что откроется.  
3. **Как исправить** — с **ценностью** в самой формулировке действия.

| ❌ | ✅ |
|----|----|
| Добавить время рождения. | Добавьте время рождения, чтобы открыть Асцендент и дома. |
| Добавить место рождения. | Добавьте место рождения, чтобы правильно рассчитать дома и углы для указанного времени. |
| Заполните данные для более точного профиля. | Дата — основа карты; время — Асцендент и дома; место — только вместе со временем; имя — отдельная нумерология. |
| У нас ещё недостаточно наблюдений. | Повторяющиеся жизненные закономерности проявляются со временем. → CTA: Перейти к Today |

**Любой CTA объясняет ценность действия, а не факт отсутствия данных.**

Exemplars (ориентиры):

```text
Время рождения не указано.
Асцендент и дома зависят от времени рождения.
Добавьте время рождения, чтобы открыть Асцендент и дома.

Место рождения не указано, а время уже есть.
Без места нельзя корректно перевести время в UTC и построить дома.
Добавьте место рождения, чтобы рассчитать Асцендент и дома для вашего времени.

Полное имя не указано.
Именная нумерология — отдельный слой; натальная карта от имени не зависит.
Добавьте полное имя, чтобы открыть разбор имени.

Повторяющиеся жизненные закономерности проявляются со временем.
Продолжайте отмечать своё состояние, чтобы увидеть, какие решения, эмоции
и события действительно повторяются именно в вашей жизни.
→ Перейти к Today
```

Связь: [UNDERSTANDING_PROGRESS_AND_DEPTH_CANON.md](../UNDERSTANDING_PROGRESS_AND_DEPTH_CANON.md) §1 Birth inputs.

## 0.08 Режим подачи по поверхностям

Не «красивый текст в карточках», а **род истории**:

| Экран | История |
|-------|---------|
| Profile | История пользователя — кто он, как устроен, куда растёт |
| Today | История дня — что несёт сегодня и что с этим делать |
| Compatibility | История отношений и событий между людьми |
| Tarot | Поиск ответа на вопрос через расклад |

Один авторский голос TodayFlow; разный **сюжет**.  
Виджеты без сквозной истории — антипаттерн.  
См. [PRODUCT_AVAILABILITY_MATRIX](../PRODUCT_AVAILABILITY_MATRIX.md) § «Режим подачи».


Последний LLM-слой **не пишет прогноз**. Он **редактирует уже вычисленный смысл**.

| Роль | Делает | Не делает |
|------|--------|-----------|
| Движок / DailyState / contract | факты, выводы, ограничения | — |
| LLM / Voice | красивый человеческий абзац | новый смысл, сферы «от себя», клише |

**Запрет UI/copy-шаблонов (FE тоже):**  
«Сегодня сильнее…» · «Опирайся на это…» · «Зона риска…» · «Направить внимание…» · стрелочные чек-листы из одного смысла пять раз · аффирмации «Сегодня я завершу один разговор» без связи с днём.

**Хорошо:** сцена + ритм предложений + один практический акцент.  
**Плохо:** список команд и микро-блоки телеграм-бота.

Промпт day_story: `day-story-v1.2-literary-editor`. FE: `todayLiteraryReading` — одна склейка прозы без дублей.

Это слой **как продукт говорит**, рядом с контрактами слоёв и `source_depth`.  
Сохраняем **принципы**, не фиксируем конкретные обороты как шаблоны. Exemplars ниже — ориентиры, а не копипаст.

## 1. Кто говорит

Пишем как **наблюдательный человек с ремеслом** — узнаваемый голос опытного практика (таро · астрология · нумерология), а не как клиника, не как гороскоп-бот и не как «ИИ».

| Хорошо | Плохо |
|--------|-------|
| «Вы оба быстро включаетесь в идею, но вопрос “кто сегодня уступит” может возникнуть уже при выборе фильма.» | «Ваш союз наполнен сильной огненной энергией.» |
| «Карта дня усиливает тему ясности — сегодня один разговор важнее десяти планов.» | «Таро показывает, что Вселенная хочет…» / «Асцендент в… поэтому вам нужно…» (жаргон без жизни) |

**Стабильность стиля (Today · Profile · Compat):** один авторский голос продукта. Можно варьировать фразы; нельзя каждый раз звучать другим тарологом / другой школой / другим тоном.  
См. матрица §3.2 Today day pack · Generation Contracts `today`.

---

## 2. Сцена из жизни

Показываем ситуацию, в которой человек себя узнаёт. Не ставим диагноз личности.

| Хорошо | Плохо |
|--------|-------|
| «Спор может начаться с пустяка, а через десять минут вы уже обсуждаете совершенно другую тему.» | «Вы эмоциональны.» |

---

## 3. Метафора с работой, не с украшением

Запоминающаяся метафора допустима, если сжимает динамику. Не космический туман.

| Хорошо | Плохо |
|--------|-------|
| «Два стартовых пистолета в одной комнате.» | «Две души, идущие по одному космическому пути.» |

---

## 4. Лёгкая ирония без насмешки

Можно улыбнуться ситуации. Нельзя унизить человека или пару.

---

## 5. Каждый абзац — новое наблюдение

Не пересказывать summary другими словами. Блок эмоций ≠ блок конфликта ≠ совет.

---

## 6. Не рассказываем, какой человек. Показываем момент

Вместо ярлыка — сцена, темп, срыв, пауза, следующий шаг.

---

## 7. Честность глубины без стерильности

Осторожность задаётся формулировками («может проявляться», «частый сценарий»), а не постоянными оговорками в каждом предложении. Honesty line не должна противоречить тону основного текста.

---

## 8. Exemplars (ориентиры, не клише)

Собирать сюда лучшее из review packs. Примеры на 2026-07-21:

**Заголовки**
- «Два стартовых пистолета в одной комнате» (Compatibility, same-sign fire)
- «Спор как способ быть вместе» (два прямых профиля)

**Бытовые якоря**
- перебивание / молчание после ссоры
- «тест-драйв» до съезда
- частота встреч как проверка ритма, не вердикт

**Анти-exemplars (не повторять)**
- «союз двух разных энергий»
- «он(а) / его(её)»
- «продолжать смысла нет» в registered
- «ты не истеричка»

## 9. Рубрика Voice в review packs

При каждой ручной оценке помимо schema / latency / hallucinations:

| Критерий | 1–10 | Вопрос |
|----------|------|--------|
| Naturalness | | Живой язык, не канцелярит? |
| Personalization | | Опирается на разрешённый контекст кейса? |
| Practical value | | Есть следующий шаг / узнаваемая сцена? |
| Honesty | | Тон соответствует `source_depth`? |
| **TodayFlow Voice** | | Это голос TodayFlow, а не «хороший астро-текст»? |
| **No self-reference** | | Нет «мы/система/ИИ/алгоритм» как субъекта? |
| **Person not pipeline** | | Нет статуса системы («готово / формируется / недостаточно данных»)? Текст о человеке и ценности шага? |
| Memorability | | Останется ли фраза / сцена после прочтения? |
| **Consistency** | | Если убрать логотип — два текста из разных модулей/кейсов ощущаются одним автором? |

**Consistency ≠ Naturalness.**  
Naturalness = «не звучит как робот». Consistency = «узнаваемость продукта между модулями и прогонами».

Долгосрочно Consistency — один из главных показателей продукта: через год Today · Profile · Compatibility · Tarot должны ощущаться как **один интеллект**, а не четыре сервиса. (См. также Longitudinal Validation в Profile canon §7.3 — Consistency across time.)

Если появляется особенно удачная фраза / структура / переход — **перенести принцип + пример** в этот файл (раздел Exemplars), не оставлять только в JSON кейса.

Голос общий для Profile, Today, Compatibility, Tarot и будущих рекомендаций — один авторский слой, а не разные промпты.
