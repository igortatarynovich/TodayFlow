## 0. TodayFlow не говорит о себе (жёсткий запрет)

> **TodayFlow не рассказывает про себя.**  
> Пользователь не читает про систему, модель, ИИ, алгоритмы или внутренние процессы.  
> Интерфейс объясняет **результат**, **ограничения** и **ценность** — субъект предложения = факт, не продукт.

Связь с [EXPLAIN_MEANING_NOT_MECHANISM.md](../EXPLAIN_MEANING_NOT_MECHANISM.md): смысл и честные границы — да; кухня и «кто посчитал» — нет.

### Запрещённые паттерны (весь продукт · UI · LLM · email · push)

| ❌ Запрещено | Почему |
|-------------|--------|
| «Мы знаем…» / «Мы уже видим…» | продукт как субъект |
| «Мы определили…» / «Мы выяснили…» | CRM / движок на сцене |
| «Система видит / знает / считает / не может…» | mechanism |
| «ИИ считает…» / «модель считает…» | ломает режим self-discovery |
| «Алгоритм…» / «На основе анализа мы…» | кухня вместо смысла |
| «Поможем тебе узнать…» от лица платформы как героя | продукт хвалит себя |

### Замены (факт / результат как субъект)

| ❌ | ✅ |
|----|----|
| Мы уже знаем ваши сильные стороны. | Основные сильные стороны уже определены. |
| Мы видим повторяющийся паттерн. | Некоторые решения повторяются достаточно часто, чтобы показать устойчивую закономерность. |
| Система не может определить асцендент. | Асцендент нельзя определить без времени рождения. |
| Мы сможем построить более точный профиль. | Время рождения позволяет определить асцендент, дома и сделать профиль точнее. |
| Система начинает узнавать тебя лучше. | С каждым закрытым днём портрет становится точнее. |

**Правило проверки:** если убрать бренд — текст всё ещё о **человеке и результате**, а не о том, кто говорит.

## 0.1 Today — literary editor (обязательно)

Последний LLM-слой **не пишет прогноз**. Он **редактирует уже вычисленный смысл**.

| Роль | Делает | Не делает |
|------|--------|-----------|
| Движок / DailyState / contract | факты, выводы, ограничения | — |
| LLM / Voice | красивый человеческий абзац | новый смысл, сферы «от себя», клише |

**Запрет UI/copy-шаблонов (FE тоже):**  
«Сегодня сильнее…» · «Опирайся на это…» · «Зона риска…» · «Направить внимание…» · стрелочные чек-листы из одного смысла пять раз · аффирмации «Сегодня я завершу один разговор» без связи с днём.

**Хорошо:** сцена + ритм предложений + один практический акцент.  
**Плохо:** список команд и микро-блоки телеграм-бота.

Промпт day_story: `day-story-v1.2-literary-editor`. FE: `todayLiteraryReading` — одна склейка прозы без дублей.

Это слой **как продукт говорит**, рядом с контрактами слоёв и `source_depth`.  
Сохраняем **принципы**, не фиксируем конкретные обороты как шаблоны. Exemplars ниже — ориентиры, а не копипаст.

## 1. Кто говорит

Пишем как **наблюдательный человек**, а не как психолог и не как астролог.

| Хорошо | Плохо |
|--------|-------|
| «Вы оба быстро включаетесь в идею, но вопрос “кто сегодня уступит” может возникнуть уже при выборе фильма.» | «Ваш союз наполнен сильной огненной энергией.» |

---

## 2. Сцена из жизни

Показываем ситуацию, в которой человек себя узнаёт. Не ставим диагноз личности.

| Хорошо | Плохо |
|--------|-------|
| «Спор может начаться с пустяка, а через десять минут вы уже обсуждаете совершенно другую тему.» | «Вы эмоциональны.» |

---

## 3. Метафора с работой, не с украшением

Запоминающаяся метафора допустима, если сжимает динамику. Не космический туман.

| Хорошо | Плохо |
|--------|-------|
| «Два стартовых пистолета в одной комнате.» | «Две души, идущие по одному космическому пути.» |

---

## 4. Лёгкая ирония без насмешки

Можно улыбнуться ситуации. Нельзя унизить человека или пару.

---

## 5. Каждый абзац — новое наблюдение

Не пересказывать summary другими словами. Блок эмоций ≠ блок конфликта ≠ совет.

---

## 6. Не рассказываем, какой человек. Показываем момент

Вместо ярлыка — сцена, темп, срыв, пауза, следующий шаг.

---

## 7. Честность глубины без стерильности

Осторожность задаётся формулировками («может проявляться», «частый сценарий»), а не постоянными оговорками в каждом предложении. Honesty line не должна противоречить тону основного текста.

---

## 8. Exemplars (ориентиры, не клише)

Собирать сюда лучшее из review packs. Примеры на 2026-07-21:

**Заголовки**
- «Два стартовых пистолета в одной комнате» (Compatibility, same-sign fire)
- «Спор как способ быть вместе» (два прямых профиля)

**Бытовые якоря**
- перебивание / молчание после ссоры
- «тест-драйв» до съезда
- частота встреч как проверка ритма, не вердикт

**Анти-exemplars (не повторять)**
- «союз двух разных энергий»
- «он(а) / его(её)»
- «продолжать смысла нет» в registered
- «ты не истеричка»

## 9. Рубрика Voice в review packs

При каждой ручной оценке помимо schema / latency / hallucinations:

| Критерий | 1–10 | Вопрос |
|----------|------|--------|
| Naturalness | | Живой язык, не канцелярит? |
| Personalization | | Опирается на разрешённый контекст кейса? |
| Practical value | | Есть следующий шаг / узнаваемая сцена? |
| Honesty | | Тон соответствует `source_depth`? |
| **TodayFlow Voice** | | Это голос TodayFlow, а не «хороший астро-текст»? |
| **No self-reference** | | Нет «мы/система/ИИ/алгоритм» как субъекта? Факт в центре? |
| Memorability | | Останется ли фраза / сцена после прочтения? |
| **Consistency** | | Если убрать логотип — два текста из разных модулей/кейсов ощущаются одним автором? |

**Consistency ≠ Naturalness.**  
Naturalness = «не звучит как робот». Consistency = «узнаваемость продукта между модулями и прогонами».

Долгосрочно Consistency — один из главных показателей продукта: через год Today · Profile · Compatibility · Tarot должны ощущаться как **один интеллект**, а не четыре сервиса. (См. также Longitudinal Validation в Profile canon §7.3 — Consistency across time.)

Если появляется особенно удачная фраза / структура / переход — **перенести принцип + пример** в этот файл (раздел Exemplars), не оставлять только в JSON кейса.

Голос общий для Profile, Today, Compatibility, Tarot и будущих рекомендаций — один авторский слой, а не разные промпты.
