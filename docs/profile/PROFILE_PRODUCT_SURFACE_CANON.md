## Главный принцип (верхний SoT)

> **Profile — это не отчёт о расчётах и не набор карточек.**  
> Это управляемое путешествие человека:  
> **узнал себя → понял почему → обнаружил закономерности → понял, куда прикладывать усилия → захотел возвращаться.**

Любой слой, блок, промпт, поле Snapshot или UI-зона проверяются:

1. **Six Questions** — [PRODUCT_BLOCK_SIX_QUESTIONS.md](../PRODUCT_BLOCK_SIX_QUESTIONS.md)  
2. **Новая ценность пути:** добавляет ли новый этап — или только повторяет уже сказанное?

Если только повторяет или нет ответа на Six Questions — **не улучшать. Удалить или объединить.**

Именно отсутствие этого уровня (путешествие человека) было причиной большинства предыдущих циклов: архитектура оптимизировала сущности системы, а не продукт, который человек покупает своим вниманием.

## 0. Что мы продаём

Не «профиль с архетипом и источниками».

Продаём ощущение:

| После Profile человек может сказать | Если не может — продукт не случился |
|-------------------------------------|--------------------------------------|
| «Да, это похоже на меня» | Есть текст, нет узнавания |
| «Теперь ясно, почему так» | Похоже на случайный AI-текст |
| «Вот этого я раньше не замечал» | Только пересказ того же ядра |
| «Понятно, куда прикладывать усилия» | Есть «миссия»-абзац без действия |
| «Хочу вернуться / проверить день» | Тупик; нет моста в Today |

**Метрика успеха — человеческая реакция, не `Hero PASS` и не `headline generated`.**

## 1. Путешествие (продукт выше секций)

Пять шагов. Это SoT смысла экрана.  
Не L0–L8. Не Snapshot. Не Gate.

```text
Шаг 1  Я чувствую, что меня поняли
Шаг 2  Я понимаю, почему выводы именно такие
Шаг 3  Я нахожу то, чего раньше не замечал
Шаг 4  Я понимаю, что делать дальше
Шаг 5  Я хочу возвращаться  →  Today · практики · карты · наблюдения · дневник
```

### Шаг 1 — Меня поняли

| | |
|---|---|
| **Что должен почувствовать** | «Ого. Это про меня.» |
| **Задача продукта** | За ≤5 секунд человек узнаёт себя |
| **Метрика** | «Да, это похоже на меня» |
| **Что на экране** | Одно имя ядра + одна сильная фраза + **большой визуальный объект** (символ / созвездие / колесо / портрет / компас). Не стена текста. Не «Личный профиль» как смысл. |
| **Share test** | Хочется отправить другу / сохранить как картинку. Если не хочется — это не Шаг 1. |
| **Не метрика** | `headline` generated · Freeze PASS · длина `identity_core` |

Пример формы (смысл, не верстка):

```text
━━━━━━━━━━━━━━━━
  НАБЛЮДАТЕЛЬ

  Ты замечаешь то,
  что остальные пропускают.
━━━━━━━━━━━━━━━━
     [ большой символ ]
```

Текст на Шаге 1 — подпись к образу. Образ первичен.

---

### Шаг 2 — Понятно, почему

| | |
|---|---|
| **Что должен почувствовать** | «А, поэтому. Это не случайный AI-текст.» |
| **Задача продукта** | Показать опоры вывода: расчётные якоря рядом со смыслом |
| **Метрика** | Человек перестаёт искать «откуда взяли» |
| **Что на экране** | Почему именно такой: Солнце · Луна · ASC (если есть) · число · другие реальные опоры — как доказательства, не как меню «Сигнатуры» |
| **Без времени рождения** | Честно без ASC/домов; что откроет время — с ценностью |
| **Новая ценность** | До Шага 2 человек знал *кто*; здесь впервые видит *почему* |

---

### Шаг 3 — Нахожу то, чего не замечал

| | |
|---|---|
| **Что должен почувствовать** | «Интересно. Вот этого я ещё не видел.» |
| **Задача продукта** | Открыть закономерность — не продолжить абзац Шага 1 |
| **Метрика** | Хотя бы один инсайт, которого не было в узнавании |
| **Форма** | **История / узел**, не три обязательные секции Strengths · Limits · Patterns |

Паттерн часто и есть главное ограничение. Тогда это **один узел**, например:

```text
Самая большая ловушка
  Ты долго наблюдаешь, прежде чем начать.
        ↓
Почему (якоря)
  Солнце…  Луна…
        ↓
Что помогает
  Начинать с первого маленького действия.
        ↓
Как это уже проявлялось  (только с living evidence)
  На прошлой неделе…
```

Это исследование личности, не четыре отдельных документа.

| Правило | |
|---------|---|
| Сильные стороны / ограничения / паттерны | **материалы** для узлов Шага 3, не обязательный список зон |
| Нет living evidence | узел без «как проявлялось»; не выдумывать повторы |
| Нет нового инсайта | не показывать «ещё список» ради заполнения |

---

### Шаг 4 — Понятно, что делать дальше

| | |
|---|---|
| **Что должен почувствовать** | «Теперь ясно, куда прикладывать усилия.» |
| **Задача продукта** | Один вектор приложения силы — не ещё одна карточка «Миссия» |
| **Метрика** | После чтения человек может сказать, *что менять в поведении / фокусе*, одним предложением |
| **Что на экране** | Направление + при необходимости короткое проявление в сферах жизни — только если это **новая** ценность после Шага 3 |
| **Не** | Абзац-пересказ «кто ты» под другим заголовком |

---

### Шаг 5 — Хочу возвращаться

| | |
|---|---|
| **Что должен почувствовать** | «Хочу проверить это в жизни / вернуться завтра.» |
| **Задача продукта** | Мост из Profile в живой продукт |
| **Куда ведёт** | Today · практики · карты · наблюдения · дневник |
| **На Profile** | Тонкий выход с ценностью («что откроется»), не day-контент внутри Profile (PR-4) |
| **Метрика** | Есть очевидный следующий шаг вне этого экрана |

Здесь начинается продукт дня. Profile его **не заменяет**.

## 2. Жёсткое правило новой ценности

Каждый следующий момент скролла обязан дать **новую** информацию:

| Реакция, которую ищем | Если вместо этого |
|-----------------------|-------------------|
| «О, вот этого ещё не видел» | перефраз предыдущего абзаца |
| «Интересно» | ещё один серый список того же |
| «Теперь понятно» / «Вот почему» | снова «кто ты», другими словами |
| «Теперь ясно, что делать» | общая фраза без вектора |

**Статья** = продолжение предыдущего текста.  
**Исследование личности** = каждый шаг открывает новый слой.

Дефект продукта (выше `PROMPT` / `UI`): секция, которая лишь продолжает предыдущий абзац.

## 3. Stop / restart

### Stop

| Stop | Why |
|------|-----|
| Следующий Freeze-блок как цель экрана | Оптимизирует сущности, не путешествие |
| Косметика карточек V2 | Маскирует отсутствие продукта |

| Любая новая зона без ответа: «какой шаг путешествия?» | Рост статьи |

### Restart

```text
1. Зафиксировать путешествие §1 (этот документ)
2. Для каждого шага — что чувствует · метрика · что на экране · откуда правда
3. Собрать механизм (поля / gates / prompts) под шаги — не наоборот
4. UI: shareable образ на Шаге 1 · доказательства на Шаге 2 · узлы на Шаге 3 · вектор на Шаге 4 · мост на Шаге 5
```

## 4. Механизм под путешествие (не продукт)

Ниже — для сборки и Freeze. **Не читать как IA экрана.**

Легенда: `HAVE` / `WEAK` / `GAP` — готовность механизма служить шагу.

### 4.1 Шаг → механизмы

| Шаг | Нужный смысл | Текущие единицы (код) | Механизм |
|-----|--------------|------------------------|----------|
| 1 Узнал себя | Имя ядра + one-liner + **визуальный объект** | archetype label · `identity_core` · pills | `WEAK`: длинный текст, нет shareable образа, нет сжатого hero-контракта |
| 2 Почему | Якоря-доказательства | astro/numerology anchors · sources strip | `WEAK`: якоря есть, не собраны как «поэтому» |
| 3 Не замечал | Узлы: ловушка / сила / повтор → why → helps → living | `strengths` · `growth_zones` · `recurring_patterns` · `helps` | `HAVE` поля; `WEAK` подача как отдельные документы; patterns gate OK |
| 4 Что делать | Один вектор усилий | `life_mission` · `life_spheres` | `HAVE` units; не путать с day agenda; не пересказ Шага 1 |
| 5 Вернуться | Мост | Maps CTA · Today journey | тонкий CTA; day-контент на Profile запрещён |

### 4.2 Истории (Шаг 3), не обязательные L3/L4/L5

| Материал | Откуда | Когда можно в узле |
|----------|--------|---------------------|
| Сила / опора | `strengths` · identity | birth готов |
| Ловушка / напряжение | `growth_zones` · patterns | birth готов; living усиливает |
| Повтор | `recurring_patterns` | **только** living gate; иначе omit |
| Что помогает | `helps` | тот же gate, что patterns |
| Почему в узле | calc якоря | всегда рядом с утверждением, не отдельной «зоной ради зоны» |
| Как проявлялось | living / maps evidence | только с evidence |

**Не требуется** три отдельные равноправные секции только потому, что в JSON три массива.

### 4.3 Hero (Шаг 1) — жёстче

| Нужно | Не нужно |
|-------|----------|
| Одно слово/имя ядра | «Личный профиль» как главный смысл |
| Одна фраза, которую узнают | Абзац-биография |
| Большой символ / колесо / созвездие / портрет / компас | Текст без объекта |
| Desire to share | Ещё pills и eyebrow |

Target-поля смысла (сжать из identity / Content Canon): имя ядра · one-liner · опционально одна reveal-строка.  
Длинный `identity_core` без сжатия = **недостаточный механизм Шага 1**, даже если Freeze identity = PASS.

### 4.4 Static / living / day

| | На Profile |
|--|------------|
| Статика (рождение) | Шаги 1–2; часть узлов Шага 3 без «прошлой недели» |
| Living | Усиливает Шаг 3 (повторы, «как проявлялось»); без living — не выдумывать |
| Ежедневно | **Не** на Profile → Шаг 5 → Today |

### 4.5 Mapping старых L0–L8 (справочно)

Бывшие L0–L8 — черновик механизма, **не** продуктовый scroll.

| Было | Куда в путешествии |
|------|---------------------|
| L0 Hero + L1 Who | Шаг 1 (слить: не Hero→Who как два абзаца статьи) |
| L2 Why | Шаг 2 |
| L3 Strengths · L4 Limits · L5 Patterns | Материалы Шага 3 (узлы / истории) |
| L6 Direction | Шаг 4 |
| L7 Evidence | Встроено в Шаг 2 и honesty; не отдельная «простыня» |
| L8 Natal | Исследуемый источник под Шагом 2 / углубление — progressive, с визуалом |
| `character_decisions` / `character_forming` | Пауза, пока не доказан новый этап путешествия |
| Freeze matrix inventory | Inventory кода; не IA продукта |

## 5. Definition of Done

Канон закрыт для перехода к UI, когда:

1. Команда проходит §1 вслух языком чувств («поняли / почему / не замечал / что делать / вернуться») — без «архетип · сигнатуры · источники» как главных слов.
2. Для Шагов 1–4 есть human sample (Case A birth-only · Case B no time · Case C with living): на каждом шаге **новая** ценность.
3. Шаг 1 проходит share test на бумаге/макете смысла (образ обязателен в спецификации, не «потом»).
4. Шаг 3 описан как узлы/истории; нет требования «три зоны потому что три массива».
5. Механизм §4 помечен HAVE/WEAK/GAP без дыр на критическом пути Шагов 1–2.
