## 0. Главный вопрос

> **Какие астрологические сущности первичные (Reference), а какие производные (Derived)?**

| Класс | Где живёт | Creation method |
|-------|-----------|-----------------|
| **Primary (atomic)** | `DATA/reference/astrology/machine/` | Manual editorial (P0.8) |
| **Composite (derived)** | runtime / SN — **не** JSON catalog | Generated by **Astrology Composition Engine** (фаза 2) |

## 3. Astrology Composition Engine (фаза 2 — backlog)

**Не P0.8.** Реализуется после P0.9 / перед или вместе с P1.0.

```
Inputs:  atomic machine records (CD)
Rules:   composition weights, orb policy, dignity modifiers (TBD)
Output:  DerivedAstroSignal (DD) — same DayModel-facing shape as §5.3 DAYMODEL
```

### 3.1 Sketch: PlanetInSign (example)

```
vector = normalize(w_p * V_planet + w_s * V_sign)     # default w_p=0.55, w_s=0.45
tempo  = merge_tempo(planet.tempo, sign.tempo)
risk_modifier = clamp(planet.risk_modifier + 0.5 * sign.risk_modifier)
confidence = min(planet.confidence, sign.confidence) * composition_discount
```

Exact weights — **Rules doc** + calibration in **P0.9** (anchor: Chariot / Year 8 / **mars + aries atoms**, not a composite file).

### 3.2 Sketch: AspectPair

```
vector = aspect_vector modulated by planet weights
risk   = f(aspect.risk, aspect.risk_modifier, tension from legacy orb)
```

## 6. CI forbidden patterns (P0.8+)

Validator **rejects** `entity_code` matching:

- `astrology.planet_in_sign.`
- `astrology.planet_in_house.`
- `astrology.transit.`
- `astrology.aspect_pair.`
- `astrology.foundation.`

Allowed prefix roots only: `astrology.sign.`, `astrology.planet.`, `astrology.house.`, `astrology.aspect.`
